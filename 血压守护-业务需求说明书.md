## 二、业务背景

### 2.1 业务问题

中国高血压患者超过2.7亿人，但高血压控制率仅为15.3%，主要原因包括：

1. **服药依从性低**：约30-40%的患者不能按时按量服药
2. **数据记录缺失**：缺乏系统化的血压数据记录
3. **预警不及时**：血压异常时无法及时发现和处理
4. **家庭参与不足**：家属无法实时了解患者健康状况

### 2.2 业务目标

| 目标编号 | 业务目标 | 衡量指标 | 目标值 |
|----------|----------|----------|--------|
| BO-01 | 提高服药依从性 | 服药按时率 | ≥80% |
| BO-02 | 完善数据记录 | 数据完整度 | ≥95% |
| BO-03 | 及时预警 | 异常发现时间 | ≤2小时 |
| BO-04 | 增强家庭参与 | 家庭功能使用率 | ≥60% |
| BO-05 | 提升用户满意度 | NPS评分 | ≥50 |

### 2.3 业务假设

| 假设编号 | 假设内容 | 验证方式 |
|----------|----------|----------|
| BA-01 | 目标用户愿意使用数字化工具管理健康 | 用户调研 |
| BA-02 | 家属有强烈意愿了解患者健康状况 | 市场调研 |
| BA-03 | 移动端是主要的交互方式 | 用户行为分析 |
| BA-04 | 免费+增值模式可行 | 转化数据分析 |

### 4.2 详细功能需求

#### FR-01：用户注册/登录

**需求描述**：
用户通过手机号注册并登录系统，建立个人账户。

**功能要点**：
- 手机号注册（短信验证码）
- 密码登录
- 忘记密码（手机验证码重置）
- 个人信息维护

**验收标准**：
- [ ] 手机号格式验证正确
- [ ] 验证码60秒重发限制
- [ ] 密码强度要求（至少8位，含字母和数字）
- [ ] 登录失败5次后锁定账户30分钟

**业务规则**：
- BR-01-01：每个手机号只能注册一个账号
- BR-01-02：验证码有效期5分钟
- BR-01-03：连续登录失败5次锁定账户30分钟

#### FR-02：药物管理

**需求描述**：
用户可以添加、编辑、删除需要服用的药物。

**功能要点**：
- 添加药物（名称、剂量、频次、时间）
- 编辑药物信息
- 删除药物（需确认）
- 查看药物列表

**验收标准**：
- [ ] 药物名称不能为空
- [ ] 剂量格式验证
- [ ] 频次选项：每日一次、每日两次、每日三次
- [ ] 时间选择支持任意时刻
- [ ] 删除操作需二次确认

**业务规则**：
- BR-02-01：每个用户最多添加50种药物
- BR-02-02：药物名称长度不超过50字
- BR-02-03：删除药物时同时删除相关服药计划

#### FR-03：服药提醒

**需求描述**：
系统根据用户设置的服药时间，按时发出提醒通知。

**功能要点**：
- 定时提醒（APP推送/短信）
- 提醒时间可自定义
- 提醒重复策略
- 勿扰模式设置

**验收标准**：
- [ ] 提醒时间误差±1分钟
- [ ] 支持多种提醒方式
- [ ] 勿扰模式生效正确
- [ ] 提醒失败有重试机制

**业务规则**：
- BR-03-01：提醒提前时间可设置（0-30分钟）
- BR-03-02：勿扰模式时间段内不发送声音提醒
- BR-03-03：提醒发出后30分钟内可确认

#### FR-04：服药确认

**需求描述**：
用户收到提醒后，可以确认已服药或标记为未服。

**功能要点**：
- 一键确认服药
- 标记为漏服
- 记录服药时间
- 查看服药历史

**验收标准**：
- [ ] 确认操作响应时间<1秒
- [ ] 支持批量确认
- [ ] 服药记录不可删除（可修正）

**业务规则**：
- BR-04-01：超过提醒时间2小时未确认，自动标记为漏服
- BR-04-02：漏服记录可手动补录
- BR-04-03：服药记录永久保存

#### FR-05：血压记录

**需求描述**：
用户可以快速记录血压数据，包括收缩压、舒张压和心率。

**功能要点**：
- 输入收缩压（70-250mmHg）
- 输入舒张压（40-150mmHg）
- 输入心率（40-200次/分）
- 添加备注
- 自动记录时间

**验收标准**：
- [ ] 数值范围验证正确
- [ ] 收缩压必须大于舒张压
- [ ] 支持手动修改时间
- [ ] 支持添加图片（测量仪照片）

**业务规则**：
- BR-05-01：收缩压正常范围90-140mmHg
- BR-05-02：舒张压正常范围60-90mmHg
- BR-05-03：心率正常范围60-100次/分
- BR-05-04：异常数值高亮显示

#### FR-06：血压趋势图

**需求描述**：
以图表形式展示血压变化趋势，帮助用户了解健康状况。

**功能要点**：
- 折线图展示血压趋势
- 支持7天/30天/90天切换
- 收缩压/舒张压双线展示
- 异常区域标注

**验收标准**：
- [ ] 图表加载时间<2秒
- [ ] 支持手势缩放
- [ ] 数据点可点击查看详细
- [ ] 图表支持导出为图片

**业务规则**：
- BR-06-01：至少2条数据才显示图表
- BR-06-02：默认显示最近7天数据
- BR-06-03：异常数值区域红色标注

#### FR-07：健康评估

**需求描述**：
基于近期血压数据，自动评估用户健康状况。

**功能要点**：
- 计算平均血压
- 评估健康等级
- 生成健康建议
- 历史对比

**验收标准**：
- [ ] 评估结果准确
- [ ] 建议内容个性化
- [ ] 评估时间可配置

**业务规则**：
- BR-07-01：基于最近7天数据评估
- BR-07-02：健康等级分四级：优秀、良好、一般、需注意
- BR-07-03：评估结果仅供参考，不替代医生诊断

#### FR-08：预警系统

**需求描述**：
实时监控用户健康状态，发现异常时发出预警。

**功能要点**：
- 漏服预警
- 血压异常预警
- 库存不足预警
- 预警分级（警告/紧急/危急）
- 预警通知（APP/短信/电话）

**验收标准**：
- [ ] 预警响应时间<5分钟
- [ ] 分级准确
- [ ] 通知送达率≥99%
- [ ] 支持预警历史记录

**业务规则**：
- BR-08-01：漏服超过2小时触发预警
- BR-08-02：收缩压≥160或≤90触发预警
- BR-08-03：舒张压≥110或≤50触发预警
- BR-08-04：药物库存不足7天触发预警

#### FR-09：家庭成员管理

**需求描述**：
用户可以添加家庭成员，实现健康数据共享。

**功能要点**：
- 添加家庭成员
- 设置家庭关系
- 授权数据访问
- 移除家庭成员

**验收标准**：
- [ ] 家庭成员数量无限制
- [ ] 授权可撤销
- [ ] 移除操作需确认

**业务规则**：
- BR-09-01：免费版最多3个家庭成员
- BR-09-02：专业版无家庭成员数量限制
- BR-09-03：被移除成员无法再访问数据

#### FR-10：协作日志

**需求描述**：
记录所有家庭成员的操作历史，保持信息透明。

**功能要点**：
- 记录所有操作
- 按时间排序
- 筛选操作类型
- 查看操作详情

**验收标准**：
- [ ] 日志记录完整
- [ ] 查询响应时间<1秒
- [ ] 支持时间范围筛选

**业务规则**：
- BR-10-01：日志记录不可删除
- BR-10-02：日志保留时间：免费版30天，专业版永久
- BR-10-03：日志包含操作人、操作类型、时间、详情

#### FR-11：数据导出

**需求描述**：
用户可以将健康数据导出为PDF报告，方便就医使用。

**功能要点**：
- 选择导出时间范围
- 生成PDF报告
- 包含数据表格和图表
- 支持分享

**验收标准**：
- [ ] PDF生成时间<5秒
- [ ] 格式美观规范
- [ ] 包含完整数据
- [ ] 支持A4纸张大小

**业务规则**：
- BR-11-01：导出时间范围不超过1年
- BR-11-02：报告包含免责声明（仅供参考）
- BR-11-03：专业版用户可导出，免费版需观看广告

#### FR-12：库存管理

**需求描述**：
自动计算药物剩余量，提醒用户及时购药。

**功能要点**：
- 设置初始药量
- 自动扣减药量
- 库存不足提醒
- 购药提醒

**验收标准**：
- [ ] 计算准确
- [ ] 提醒及时
- [ ] 支持手动调整

**业务规则**：
- BR-12-01：每次确认服药自动扣减1次剂量
- BR-12-02：剩余药量不足7天时提醒
- BR-12-03：支持手动设置实际药量
