### 2.3 ICP备案流程

**重要提示**：在中国大陆境内提供网站服务必须完成ICP备案，否则网站将被封锁。

#### 备案前置条件
- [ ] 已完成域名实名认证（超过3天）
- [ ] 已购买大陆境内云服务器（包年包月≥3个月）
- [ ] 准备身份证正反面照片
- [ ] 准备网站负责人照片（幕布拍照或APP人脸验证）

#### 阿里云备案流程（约7-20个工作日）

**阶段1：填写备案信息（1-2天）**
1. 登录阿里云控制台
2. 导航到"备案" -> "ICP备案"
3. 点击"开始备案"
4. 填写主体信息：
   - 主体类型：个人 或 企业
   - 证件号码：身份证/营业执照
   - 通讯地址：详细住址
   - 手机号码：负责人手机
5. 填写网站信息：
   - 网站名称：血压守护（个人备案建议用"XX的个人网站"）
   - 域名：已注册的域名
   - 服务内容：综合性网站
   - 备注：血压记录与健康管理系统

**阶段2：阿里云初审（1-2个工作日）**
- 阿里云客服会电话核实信息
- 如有问题会退回修改
- 通过后进入管局审核

**阶段3：管局审核（7-20个工作日）**
- 各省通信管理局审核
- 审核期间网站不能访问
- 审核结果通过短信和邮件通知

**阶段4：备案成功**
- 获得ICP备案号（如：京ICP备2024XXXXXX号）
- 需要在网站底部展示备案号
- 备案号链接到工信部网站

#### 各省备案时长参考
| 省份 | 预计时长 | 备注 |
|------|----------|------|
| 北京 | 7-14个工作日 | 较快 |
| 上海 | 7-14个工作日 | 较快 |
| 广东 | 10-20个工作日 | 稍慢 |
| 浙江 | 7-15个工作日 | 正常 |
| 江苏 | 10-18个工作日 | 正常 |

2. 导航到"产品" -> "SSL证书"

### 3.5 Nginx HTTPS配置

创建HTTPS配置文件：

```nginx
# /etc/nginx/sites-available/bp-guardian-ssl

server {
    listen 80;
    server_name your-domain.com www.your-domain.com;
    
    # HTTP自动跳转HTTPS
    return 301 https://$server_name$request_uri;
}

server {
    listen 443 ssl http2;
    server_name your-domain.com www.your-domain.com;
    
    # SSL证书配置
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    
    # SSL优化配置
    ssl_protocols TLSv1.2 TLSv1.3;
    ssl_ciphers HIGH:!aNULL:!MD5;
    ssl_prefer_server_ciphers on;
    ssl_session_cache shared:SSL:10m;
    ssl_session_timeout 10m;
    
    # 安全响应头
    add_header Strict-Transport-Security "max-age=63072000; includeSubDomains" always;
    add_header X-Frame-Options "SAMEORIGIN" always;
    add_header X-Content-Type-Options "nosniff" always;
    add_header X-XSS-Protection "1; mode=block" always;
    
    # 静态文件缓存
    location /static {
        alias /var/www/blood-pressure-guardian/app/static;
        expires 30d;
        add_header Cache-Control "public, immutable";
    }
    
    # 前端文件
    location / {
        root /var/www/blood-pressure-guardian/www;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
    
    # API反向代理
    location /api {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 120s;
        proxy_read_timeout 120s;
        proxy_send_timeout 120s;
    }
    
    # 禁止访问隐藏文件
    location ~ /\. {
        deny all;
    }
}
```

#### 启用配置
```bash
# 创建软链接
sudo ln -s /etc/nginx/sites-available/bp-guardian-ssl /etc/nginx/sites-enabled/

# 测试配置
sudo nginx -t

# 重启Nginx
sudo systemctl restart nginx

# 验证HTTPS
curl -I https://your-domain.com
```

创建自动备份脚本 `/var/www/blood-pressure-guardian/backup.sh`：

```bash
#!/bin/bash
# 数据库备份脚本

BACKUP_DIR="/var/www/blood-pressure-guardian/backups"
DATE=$(date +%Y%m%d_%H%M%S)
DB_NAME="bp_guardian"
DB_USER="bp_user"
DB_PASS="your_password"

# 创建备份目录
mkdir -p $BACKUP_DIR

# 备份数据库
mysqldump -u $DB_USER -p$DB_PASS $DB_NAME > $BACKUP_DIR/bp_guardian_$DATE.sql

# 压缩备份文件
gzip $BACKUP_DIR/bp_guardian_$DATE.sql

# 删除7天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

echo "备份完成: bp_guardian_$DATE.sql.gz"
```

设置定时任务：
```bash
# 编辑crontab
crontab -e

# 添加每天凌晨2点备份
0 2 * * * /var/www/blood-pressure-guardian/backup.sh >> /var/www/blood-pressure-guardian/logs/backup.log 2>&1
```
